PLAYOFFSjs
takes your data and draws a tree of a knockout tournament
CreatePlayoffs()

It's a single entry point to Playoffs. 

import { createPlayoffs } from 'playoffs'

const lifecycle_methods = createPlayoffs(data, wrapper, options)

Parameters
  • Data (required): a data of certain shape that describes rounds and matches
  • Wrapper element (required): an element in which playoffs' markup will be installed.
  • Options (optional)
Return value
  • An object containing lifecycle methods:
    • moveToPreviousRound
    • moveToNextRound
    • moveToLastRound
    • setBaseRoundIndex
    • getNavigationState
    • applyNewOptions
    • getUserOptions
    • replaceData
    • applyMatchesUpdates
    • getAllData
    • highlightContestantHistory
    • uninstall

You can call it many times

You may install as much Playoffs on your page as you like.
It's also possible to call createPlayoffs() with the same wrapper element twice, in such case a new instance will be installed in place of an old one. Though doing so makes little sense. If you need to switch to another tournament, better use replaceData method instead of calling createPlayoffs() again.

About wrapper element

In most cases it makes sense to set explicit height on your wrapper element and keep "width" and "height" options at a default of '100%'. It will help prevent a height jump when playoffs are installed
