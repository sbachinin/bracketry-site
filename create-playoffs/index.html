PLAYOFFSjs
takes your data and draws a tree of a knockout tournament
CreatePlayoffs()

It's a single entry point to Playoffs. 

import { createPlayoffs } from 'playoffs'

const lifecycle_methods = createPlayoffs(data, wrapper, options)

Parameters
  • Data (required): a data of certain shape that describes rounds and matches
  • Wrapper element (required): an element in which playoffs' markup will be installed.
  • Options (optional)
Return value
  • An object containing lifecycle methods:
    • moveToPreviousRound
    • moveToNextRound
    • moveToLastRound
    • setBaseRoundIndex
    • getNavigationState
    • applyNewOptions
    • getUserOptions
    • replaceData
    • applyMatchesUpdates
    • getAllData
    • highlightContestantHistory
    • uninstall

You can call it many times

You may install as much Playoffs on your page as you like. It's also possible to call createPlayoffs() with the same wrapper element twice, in such case a new instance will be installed in place of an old one. Though doing so makes little sense. If you need to switch to another tournament, better use replaceData method instead of calling createPlayoffs() again.

Problem of height

In most cases it makes sense to set explicit height on your wrapper element and keep "width" and "height" options at a default "100%". You can also provide an auto-sized wrapper and set more explicit width and height options. But in such case you are likely to get a height jump during playoffs' installation. If you don't provide any heights (neither on your wrapper nor in playoffs' options), playoffs will expand to the full height of its content. If this is what you need, mind one little problem: "full height" is not stable, i.e. it will change when you navigate to later rounds (it will get twice smaller with every next round). Thus all content after playoffs will be pushed back and forth. So if you really want this auto height, a solution to the problem of jumping height will be useClassicalLayout option. When set to true, it guarantees that the height will remain stable.
